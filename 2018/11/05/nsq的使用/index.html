nsq的使用

刘洋的博客
2018-11-05
学习

下载

nsq package

配置

vim /usr/lib/systemd/system/nsqlookupd.service

vim /usr/lib/systemd/system/nsqd.service

vim /usr/lib/systemd/system/nsqadmin.service

Type没有值时就注释掉，不然注册服务会失败。 头[unit]一定要加，不能省，不然注册服务会失败

注册

查看服务是否开启

其他

如果关闭了防火墙，就忽略
防火墙添加4150、4171，其中4171是web监控端口

（–permanent永久生效，没有此参数重启后失效）
重新载入

查看

启动之后，通过 http://127.0.0.1:4171/ 可以访问nsq管理页面

使用

官方包

发布

我们可以查看下nsq的管理平台，可以看到Nsqd接收到了10条信息，10条信息都储存在内存中，没有被消化。没有任何服务订阅了这个topic，所以topic的消息都没有被消化。
